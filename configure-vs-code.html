This step is pretty long. Ask for help if you run into any trouble!
  1. VS Code should start up and show you a welcome screen. You can customize the theme and explore the other new user suggestions if you like, but you can ignore or close that whole “Getting Started” tab whenever you like.

  2. Click the Extensions icon in the sidebar on the left. It is the one that has three connected squares, plus a tilted fourth square floating in the upper right:

  3. Inside the extension list that appears, type Java in the “Search Extensions” box:

  4. One of the results, probably the very first one, will be the Extension Pack for Java from Microsoft. The name may be truncated, as in the picture below:

    ...but if it is, you can click the extension to see the full details and make sure you have the right one:

    ⚠️ CAUTION Be sure you getting Java, not JavaScript! The two are very different languages that just happen to have similar names.

    (The latter was originally called “LiveScript,” but they changed the name to “JavaScript” at the last second because they heard that something big named “Java” was coming out soon, and they wanted to get in on the action. See Wikipedia on the history of JavaScript for more details )

    Both languages are useful and widely used, and you can install both in VS Code if you want to. But Java is the one you will need for this class.

  5. If you can’t find the Java extension pack, then try this link and click the big Install button.
  6. Once you’re sure you have the correct extension, click Install. Installation may take a while. Be patient.
  7. You may get a welcome screen that encourages you to install a JDK. You shouldn’t need to do that; you already installed it earlier in these instructions.
  8. To verify that the installation worked, press ⌘-shift-P (Mac) / ctrl-shift-P (Windows) to open Visual Studio’s “command palette,” which lets you search for text editor commands by name. Type java into the command palette. You should see a whole bunch of commands whose name starts with “Java:”

    You might not see exactly the same commands as the one in the screenshot. That is OK! Just make sure there is a large number of commands that start with “Java:”, whatever the commands are.

  9. You don’t need to run any of those commands right now; you are just checking that they’re there. Press esc on your keyboard to exit the command palette.
  10. If you don’t see all those Java commands, then:
    1. Try quitting and reopening Visual Studio Code again and repeat step 6.
    2. If that doesn’t work, go back to step 4 and try it again.
    3. If that still doesn’t work, ask the instructor or a preceptor for help.
  11. Open the VS Code settings by pressing ⌘-comma (Mac) or ctrl-comma (Windows). If the shortcut isn’t working, then navigate to Code menu → Preferences → Settings (Mac) / File menu → Preferences → Settings (Windows). There are several things to change in the settings:

    1. Search for the setting named “Auto Save” using the “Search settings” box. Set the dropdown to onFocusChange. This way, every time you click the run button, it will save your code first.

    2. We want to disable “hybrid mode,” which is a temperamental VS Code option that sometimes causes projects to not run or load properly. Search for “launch mode” in the settings search box, then set Java > Server: Launch Mode to Standard:

    3. Finally, search for “generatesMetadataFilesAtProjectRoot” in the settings search box (you can copy and paste that), and make sure that “Generates Metadata Files At Project Root” is enabled (i.e. the checkbox is checked):

    4. After changing that last setting, you may get this message in the lower right corner: 

      ⚠️ Java Language Server configuration changed, please restart Visual Studio Code.
Source: Language Support for Java(TM) by Red Hat...

      Click “Restart Now.” (Don’t worry, VS Code should restart quickly.) If you don’t see this message, quit and reopen VS Code just to be safe.

🎉 You did it! Marvel at your nicely-configured installation of VS Code, then proceed to install the git version control system.
